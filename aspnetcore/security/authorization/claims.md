## <a name="adding-claims-checks"></a><span data-ttu-id="58e24-113">Agregar comprobaciones de notificaciones</span><span class="sxs-lookup"><span data-stu-id="58e24-113">Adding claims checks</span></span>

<span data-ttu-id="58e24-114">Las comprobaciones de autorización basadas en notificaciones son declarativas: el desarrollador las inserta en el código, en un controlador o en una acción dentro de un controlador, especificando las notificaciones que el usuario actual debe poseer y, opcionalmente, el valor que debe tener la notificación para tener acceso al recurso solicitado.</span><span class="sxs-lookup"><span data-stu-id="58e24-114">Claim based authorization checks are declarative - the developer embeds them within their code, against a controller or an action within a controller, specifying claims which the current user must possess, and optionally the value the claim must hold to access the requested resource.</span></span> <span data-ttu-id="58e24-115">Los requisitos de las notificaciones están basados en directivas, el desarrollador debe compilar y registrar una directiva que exprese los requisitos de las notificaciones.</span><span class="sxs-lookup"><span data-stu-id="58e24-115">Claims requirements are policy based, the developer must build and register a policy expressing the claims requirements.</span></span>

<span data-ttu-id="58e24-116">El tipo más sencillo de la Directiva de notificaciones busca la presencia de una demanda y no comprueba el valor.</span><span class="sxs-lookup"><span data-stu-id="58e24-116">The simplest type of claim policy looks for the presence of a claim and doesn't check the value.</span></span>

<span data-ttu-id="58e24-117">En primer lugar, debe compilar y registrar la Directiva.</span><span class="sxs-lookup"><span data-stu-id="58e24-117">First you need to build and register the policy.</span></span> <span data-ttu-id="58e24-118">Esto se produce como parte de la configuración del servicio de autorización, que normalmente participa en `ConfigureServices()` el archivo *Startup.CS* .</span><span class="sxs-lookup"><span data-stu-id="58e24-118">This takes place as part of the Authorization service configuration, which normally takes part in `ConfigureServices()` in your *Startup.cs* file.</span></span>

::: moniker range=">= aspnetcore-3.0"

```csharp
public void ConfigureServices(IServiceCollection services)
{
    services.AddControllersWithViews();
    services.AddRazorPages();

    services.AddAuthorization(options =>
    {
        options.AddPolicy("EmployeeOnly", policy => policy.RequireClaim("EmployeeNumber"));
    });
}
```

::: moniker-end

::: moniker range="< aspnetcore-3.0"
